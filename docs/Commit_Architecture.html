02   Three Extraction Trigger Paths

Every signal source is checked via three independent paths — ensuring commitments arrive within the SLA regardless of whether the user is active, inactive, or an event just occurred.

⚡ Path 1 — Webhook (Immediate)

Latency: < 30 seconds · Triggered by Graph change notifications
Graph sends POST to /api/v1/webhook
HMAC-SHA256 signature validated
clientState anti-replay check
subscriptionId → userId (TokenCache)
ExtractionOrchestrator.ExtractAndStoreAsync()
TokenCache.MarkExtracted(userId)

⏰ Path 2 — Background Polling (5 min)

Latency: ≤ 5 minutes · Ensures D-001 metric even without webhooks
ExtractionPollingService wakes (PeriodicTimer)
TokenCache.GetActiveUserIds() → eligible list
Filter: token valid + ≥ 3 min since last extract
Task.WhenAll — all eligible users concurrently
ExtractionOrchestrator.ExtractAndStoreAsync()
MarkExtracted per user, retry next cycle on failure

👤 Path 3 — User-Initiated (Fire & Forget)

Latency: immediate · Triggered on pane open or explicit refresh
User opens Teams pane → GET /commitments/{userId}
Bearer token stored → TokenCache.Store(userId, token)
Check: LastExtracted > 3 min ago?
Task.Run (fire-and-forget) — response not blocked
ExtractionOrchestrator runs in background
POST /extract for explicit user-initiated refresh

06   Production-Grade Evidence

🔒 Security

✓HMAC-SHA256 on every webhook — CryptographicOperations.FixedTimeEquals (timing-safe)
✓clientState anti-replay — every Graph notification validated before processing
✓OBO token scoping — only accesses data the signed-in user has delegated rights to
✓JWT expiry parsing — TokenCache never serves an expired token; ConcurrentDictionary thread-safe
✓PII scrubbing — all logs hash userId with SHA-256 (PiiScrubber.HashValue)
✓Right-to-erasure endpoint — DELETE /users/{userId}/data removes all data + Graph subscriptions

⚡ Concurrency & Resilience

✓Task.WhenAll for all 4 extractors — no sequential blocking, full concurrent I/O
✓Fire-and-forget extraction — GET /commitments never blocked by extraction latency
✓PeriodicTimer (not Timer) — avoids timer drift, graceful CancellationToken teardown
✓Per-user extraction guard — 3-minute debounce prevents duplicate concurrent runs per user
✓Typed CommitException — all errors become structured HTTP responses with requestId
✓Webhook dedup — HashSet per batch prevents duplicate extraction per user per notification

07   Psychology & Motivation Layer (P-27)

Behavioral science is first-class infrastructure — not a UI afterthought. 14 frameworks implemented in the constitution; 6 of them wired into live components.

🎯 Self-Determination Theory

Competency levels (Novice→Expert) grow with delivery score. Users see progress, not tasks.

📈 Progress Principle

DeliveryScore with previous-period comparison. StreakDays displayed. Small wins surface.

⚠️ Loss Aversion

Cascade view frames delay as "commitments at risk" — loss framing drives action faster than gain framing.

🔗 Fogg Behavior Model

1-click approval minimizes friction at peak motivation. ApprovalCard.tsx is 1 tap.

🌊 @react-spring/web

Stagger animations on cascade items. Progress bar springs. Micro-animations signal causality.

🎰 Variable Ratio Reinforcement

CascadeHealthRate + XP grant unpredictable rewards for on-time closes. Habit-loop inducing.

🏁 Completion Bias

Eisenhower priority labels + impact score make "what to finish next" unambiguous — reduces open-loop anxiety.

🧩 Autonomy Preservation

Approve / Edit / Skip — the agent always asks, never acts. Human is in the loop for every action.
