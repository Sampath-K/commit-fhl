Phase 1 — The Idea
The pain point that started it all
Human
"One of the draining tasks every engineer has is follow-ups. The one who has the task may forget and the one who assigned may forget too or get frustrated with pings to check if there are updates."

Not a feature request. A real observation about how engineering teams silently lose trust and velocity through follow-up friction.

Scope expanded: every signal, not just one
Agent

Proposed capturing commitments from all M365 surfaces: meeting transcripts, Teams chat, email, ADO threads, document comments — anywhere a task silently hides.

The cascade insight
Human
"Can you also include impact to dependent tasks? Waiting on someone is one of the reasons for silent slips. The impact of not getting it done should be crystal clear."

This one prompt transformed the idea from a task tracker into a dependency-aware cascade engine — something qualitatively different from anything in M365 today.

The emotional north star defined
Human
"The goal should be to reduce burnouts and increase the joy of getting this done without the fear of anything slipping through the cracks. Users should believe in the system."

This became the soul of the product: not productivity tooling, but psychological safety for engineers.

The resumability problem raised
Human
"I do not want context to get full and the need for me to give instructions again. Can we get this to a solid state that can be resumed in case of failures and not restarted?"

This forced a structural solution: not just planning, but a session state system that any agent can cold-start from.

One consistent team pattern requested
Human
"I want this to be a pattern for the team to follow so one consistent recommendation is what I want to give."
Full project scaffold generated
Agent

Blocker #2 — jq not installed on this machine
Blocker
.agents/commit-fhl/setup-env.sh: line 161: jq: command not found

The initial script used jq to parse Azure CLI JSON output. Not installed, not worth installing just for this.

Fix: Rewrote script using az --query throughout
Fix

Every jq -r '.field' replaced with az ... --query "field" -o tsv. Azure CLI's built-in JMESPath query engine does everything jq was needed for. Script now has zero external dependencies.
